WhatsApp API Documentation

Developer Guide & Server Capacity Information

Overview

This API allows you to send and receive WhatsApp messages programmatically. It is built on top of Baileys library and runs on a optimized Node.js server.

Important: This is an unofficial API. Use responsibly to avoid number bans.

Usage Constraints & Capacity

The server is hosted on a Digital Ocean Droplet with the following specifications:

  • CPU: 1 vCPU
  • RAM: 2 GB
  • Storage: 70 GB NVMe

Estimated Capacity

Text Messages

~50-100 messages / second

High throughput

Media (Images/Video)

~5-10 uploads / second

Bandwidth limited

Concurrent Sessions

~5-10 active sessions

RAM limited (each session consumes ~100MB)

Rate Limits

Global API Rate Limit: 100 requests per 15 minutes per IP.

API Endpoints

Authentication

POST /auth/login
{
  "email": "user@example.com",
  "password": "password"
}
-> Returns { token, user }

Sending Messages

Headers: x-api-key: YOUR_API_KEY

POST /messages/send-text
{
  "sessionId": "marketing",
  "to": "1234567890",
  "text": "Hello World!"
}
POST /messages/send-image
{
  "sessionId": "marketing",
  "to": "1234567890",
  "url": "https://example.com/image.png",
  "caption": "Check this out!"
}

Session Management

Headers: Authorization: Bearer JWT_TOKEN

GET /api/admin/stats
GET /sessions
